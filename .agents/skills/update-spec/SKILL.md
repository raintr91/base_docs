# /update-spec — Controlled spec delta

**Extracts:** `extractBundle: update-spec` → `.cursor/extracts/extract-registry.json`

Doc hub: `base-docs/platform/toolchain/UPDATE-SPEC-FLOW.md` · `base-docs/platform/toolchain/FEATURE-ARTIFACT-FLOWS.md`

## Load policy

| Load | Do not load |
|------|-------------|
| `base-docs Code / `--id`` — **SSOT** patch target | `ir/*.yaml` — regenerated by split |
| `base-docs Code / `--id`` — E2E testcase deltas | Legacy source, codegen manifests |
| `spec-update-delta.md` matrix | Full bundle rewrite |

## Scope

**In:** patch bundle (+ testcase when E2E scope changes); `pnpm spec:split`; emit `#update:*`; bump `specRevision`.

**Out:** full rewrite (`/spec`), legacy re-mine (`/update-spec-legacy`), production code, direct `ir/` edits.

## Workflow

1. Identify delta scope (one scenario / block / API field).
2. Patch minimal YAML sections in **bundle** per `spec-update-delta.md` (not `ir/*`).
3. Emit matching `#update:*` tags; bump `specRevision`.
4. If `featureStatus` was `wire` → `need-update`.
5. Record harness notes when present.
6. `pnpm spec:split -- base-docs Code / `--id``
7. `pnpm spec:split:check -- base-docs Code / `--id``
8. User runs `pnpm docs:render` (manual).
9. Follow-up per matrix: `portal:gen [--force]`, `/bqa-grill-docs`, `/dev-grill-docs`, `/test`, or `/wire`.
